## 긴 컨텐츠 / 스크롤 처리

배경:
- 현재 리더는 `현재 보이는 viewport OCR` 중심이다.
- 스크롤이 필요한 긴 문서는 한 번의 OCR/Assist로 안정적으로 다루기 어렵다.

방향:
- 자동 스크롤보다 `수동 스크롤 + 누적 읽기 세션`을 우선한다.
- 기본 `Assist`는 계속 현재 화면 기준으로 유지한다.
- 긴 문서용으로 누적 읽기 흐름을 별도로 추가한다.

후보 작업:
- `추가 읽기` 액션 추가
- `세션 초기화` 액션 추가
- 스크롤 후 새 OCR 결과를 기존 세션에 누적
- 이전 스냅샷 말미와 현재 스냅샷 시작 부분을 비교해 중복 줄 제거
- `현재 화면 기준 도움`과 `누적 내용 기준 도움`을 분리

이유:
- 앱마다 스크롤 동작과 레이아웃이 달라 자동 스크롤은 깨지기 쉽다.
- sticky header, lazy loading, 광고, 포커스 이동 같은 변수가 많다.
- 수동 스크롤 + 누적 세션이 구현 난이도 대비 안정성이 높다.
